# 11 后续能力追踪文档

版本：v0.1  
日期：2026-06-14  
状态：非 MVP 长期能力基线

## 1. 文档定位

本文档用于承接原始产品方案中已经明确提出、但未进入 MVP 的长期能力。

这些能力不是被删除，也不是被否定，而是基于“MVP 先验证推荐准确和用户价值”的原则，暂时不进入 `RR-MVP-*` 范围。后续任何长期能力进入开发前，必须先从 `RR-FUTURE-*` 升级为具体阶段需求编号、接口、数据模型、测试项和验收标准。

本文件不得扩大当前 MVP 范围。MVP 仍以研究画像、开放数据源检索、跨源排重、个性化推荐、AI 分析、用户反馈、知识库基础能力、日报/周报和成本控制为第一目标。

## 2. 编号规则

长期能力编号格式：`RR-FUTURE-数字`。

长期能力进入实现前必须补齐：

- 阶段归属。
- 用户场景。
- 产品范围。
- 技术方案。
- API 或接口变化。
- 数据模型变化。
- 测试和验收标准。
- 合规限制。

## 3. 后续能力清单

### RR-FUTURE-001 多渠道推送

目标：

- 在邮件和 Web 消息中心之外，支持微信、企业微信、飞书、钉钉和 App 通知。

触发时机：

- Phase 5 增长与商业化阶段。
- 当日报、周报、公开内容和团队协作需要更高触达率时启动。

前置条件：

- 日报和周报内容质量稳定。
- 用户通知偏好、退订、频率控制已经可用。
- 消息模板和发送状态已结构化。

验收标准：

- 用户可按项目和报告类型配置通知渠道。
- 每条通知记录发送状态、失败原因、重试状态和打开行为。
- 用户可退订或关闭任一渠道。
- 多渠道推送失败不影响站内消息生成。

合规限制：

- 不得未经用户同意向外部渠道推送研究内容。
- 企业微信、飞书、钉钉等组织渠道必须遵守组织授权和应用权限边界。

### RR-FUTURE-002 月报系统

目标：

- 在日报和周报基础上，生成月度科研趋势和课题推进报告。

触发时机：

- Phase 2 可评估基础版本，但不得阻塞全文研读。
- Phase 3 后可结合结构化知识库增强质量。

前置条件：

- 日报和周报稳定生成。
- 推荐、收藏、研读、反馈和知识库增长数据可按月聚合。

验收标准：

- 月报包含研究趋势、技术路线变化、竞争团队、高价值论文、研究空白变化、用户课题推进情况和下一阶段建议。
- 月报区分事实、AI 归纳、多文献对比、AI 推测和研究启发。
- 月报中的关键结论可回溯到论文、推荐批次、知识库条目或用户行为。

合规限制：

- 不得在公开月报中展示用户私有研究材料、导师意见或未公开实验记录。

### RR-FUTURE-003 中文学术数据源

目标：

- 补充中文期刊、学位论文和本土研究成果，提高中文科研场景覆盖。

触发时机：

- Phase 3 数据源扩展阶段。
- 当目标用户明确需要中文文献、中文学位论文或本土研究成果时启动。

前置条件：

- 开放数据源适配器架构稳定。
- 数据源合规审查流程可用。
- 中文标题、摘要、关键词和作者机构字段标准化规则可用。

验收标准：

- 中文数据源通过官方 API、正式授权、用户导出或合规合作接入。
- 中文文献可进入统一 SourceRecord、PaperVersion、Paper、Recommendation 流程。
- 中文文献与英文文献可跨语言排重或关联。
- 中文文献推荐解释可说明命中关键词、方法、材料和研究对象。

合规限制：

- 不实现全自动知网登录。
- 不云端保存学校账号、图书馆密码、统一认证密码或长期 Cookie。
- 不绕过商业数据库访问限制。

### RR-FUTURE-004 用户自定义来源

目标：

- 允许用户配置期刊官网、作者主页、实验室主页、会议网站、RSS、机构知识库和公开专题页。

触发时机：

- Phase 3 数据源扩展阶段。
- 当用户需要跟踪特定期刊、作者、实验室或会议时启动。

前置条件：

- Source Adapter 插件化结构稳定。
- 抓取频率、缓存、失败降级和合规策略可配置。
- 用户自定义来源需要具备审核或安全限制。

验收标准：

- 用户可添加、编辑、暂停和删除自定义来源。
- 每个来源记录类型、URL、抓取频率、上次抓取时间、失败状态和合规说明。
- 自定义来源产出的记录进入统一标准化、排重和推荐流程。
- 来源失败不会影响其他检索任务。

合规限制：

- 不抓取需要绕过登录、验证码、付费墙或访问控制的内容。
- 对不允许自动抓取的网站必须停止接入。

### RR-FUTURE-005 学科专业数据库

目标：

- 接入医学、生命科学、人工智能、材料、环境等专业数据库，增强垂直学科覆盖。

触发时机：

- Phase 3 数据源扩展阶段。
- 当首批学科之外的用户增长或付费需求明确时启动。

前置条件：

- 通用开放数据源已经稳定。
- 每个专业数据库有清晰字段映射和许可策略。
- 推荐评测集扩展到对应学科。

验收标准：

- 专业数据库适配器输出统一 SourceRecord 字段。
- 数据源质量评分参与推荐排序。
- 对专业标识符、主题词、医学词表或学科分类进行保留。
- 至少完成一个垂直学科的端到端推荐评测。

合规限制：

- 遵守对应数据库 API 条款、许可和限流要求。
- 不将受限数据库内容公开再分发。

### RR-FUTURE-006 移动端能力

目标：

- 提供 App 或 PWA，用于推送查看、快速反馈、收藏、稍后阅读和轻量研读。

触发时机：

- Phase 4 本地连接器与多端能力阶段。
- 当日报、周报和推荐反馈需要移动场景提升留存时启动。

前置条件：

- Web 端推荐、反馈、消息中心和知识库基础能力稳定。
- API 权限、分页、离线状态和通知设置可复用。

验收标准：

- 用户可在移动端查看日报、周报和推荐列表。
- 用户可完成收藏、稍后阅读、不相关、方法可借鉴等快速反馈。
- 移动端不承担复杂全文研读编辑，默认跳转 Web 或异步任务。
- App 或 PWA 通知尊重用户通知偏好。

合规限制：

- 移动端不得缓存未授权全文。
- 本地缓存必须可清理。

### RR-FUTURE-007 高级增长资产

目标：

- 建立可传播的公开内容资产，包括公开方向周报、文献时间线、组会文献包和公开研读报告。

触发时机：

- Phase 5 增长与商业化阶段。
- 当个人知识库和 AI 研读质量稳定后启动。

前置条件：

- AI 研读报告具备事实分级和来源追溯。
- 用户可控制公开范围。
- 分享页面具备基础访问权限和撤回能力。

验收标准：

- 用户可生成公开只读研读报告。
- 用户可生成研究方向公开周报。
- 用户可生成主题文献时间线。
- 用户可生成组会文献包，包含阅读清单、参数对比、讨论问题和二维码或分享链接。
- 未注册用户可查看公开基础内容，注册后可复制到个人知识库。

合规限制：

- 公开内容不得包含用户私有上传文件全文、未公开实验记录或敏感导师意见。
- 原文引用必须短且用于证据定位。

### RR-FUTURE-008 企业与机构集成

目标：

- 支持企业微信、飞书、钉钉组织空间、团队权限、审计和团队推送，服务课题组、高校机构和企业研发团队。

触发时机：

- Phase 6 课题组与机构版阶段。

前置条件：

- 个人版研究项目、知识库、成本、审计和推送体系稳定。
- 团队空间、成员角色和共享知识库模型明确。

验收标准：

- 支持组织空间、成员邀请、角色权限和项目共享。
- 支持团队额度、成员成本统计和审计日志。
- 支持组织渠道推送日报、周报、任务和阅读提醒。
- 导师或管理员可查看团队阅读、收藏、研读和知识沉淀质量。

合规限制：

- 企业和机构空间必须具备数据隔离。
- 管理员可见范围必须清晰，不得默认查看成员私有项目。

### RR-FUTURE-009 高级数据源合规策略

目标：

- 为商业数据库授权、机构合作、用户本地权限连接器和私有部署建立合规策略。

触发时机：

- Phase 4 本地权限与多端能力阶段开始建设。
- Phase 6 机构与私有部署阶段完善。

前置条件：

- 检索与全文获取分离原则已在 MVP 落地。
- 文件上传、对象存储、审计和权限控制稳定。

验收标准：

- 商业数据库只通过正式授权、机构合作或用户主动导出接入。
- 用户本地权限连接器只接收用户合法获取并主动授权的元数据和文件。
- 平台不保存学校账号、图书馆密码、统一认证密码或长期原始 Cookie。
- 所有全文来源记录许可、授权方式、获取时间和用户授权状态。

合规限制：

- 不绕过机构权限。
- 不批量下载或分发非开放全文。
- 不将本地权限能力包装为云端代下服务。

### RR-FUTURE-010 后期受控 Agent

目标：

- 在产品成熟后引入受控专业 Agent，提高检索规划、论文分析、证据定位、研究空白和实验比较能力。

触发时机：

- 产品成熟后。
- 只有当确定性工作流已稳定且单点 AI 任务不足以满足复杂科研推理时启动。

前置条件：

- 工作流引擎、任务状态、成本记录、审计日志和失败降级稳定。
- Agent 输入输出 schema 明确。
- 有可回放评测集和人工审核机制。

验收标准：

- Query Planner Agent 能生成可审计检索计划。
- Paper Analysis Agent 能输出结构化研读结果。
- Evidence Agent 能定位证据并标注可信度。
- Research Gap Agent 能生成带来源的研究空白候选。
- Experiment Comparison Agent 能比较参数、方法和性能差异。
- 所有 Agent 均由工作流调度，任务可超时、可终止、可重试、可审计。

合规限制：

- 不允许无限自主循环。
- 不允许绕过额度、权限、证据校验和人工审核。
- 不允许将 AI 推测伪装成原文事实。

## 4. 阶段映射

| 阶段 | 后续能力 | 说明 |
| --- | --- | --- |
| Phase 2：全文研读 | `RR-FUTURE-002` | 可评估月报基础版，但不得阻塞全文研读 |
| Phase 3：科研知识系统 | `RR-FUTURE-003`、`RR-FUTURE-004`、`RR-FUTURE-005` | 扩展中文、用户自定义和学科专业数据源 |
| Phase 4：本地连接器 | `RR-FUTURE-006`、`RR-FUTURE-009` | 建设移动端、本地权限和合规连接策略 |
| Phase 5：增长与商业化 | `RR-FUTURE-001`、`RR-FUTURE-007` | 增强触达渠道和公开增长资产 |
| Phase 6：课题组与机构版 | `RR-FUTURE-008` | 建设团队、企业和机构集成 |
| 产品成熟后 | `RR-FUTURE-010` | 引入受控 Agent，不允许无限自主循环 |

## 5. 升级为开发需求的规则

任一 `RR-FUTURE-*` 进入开发前，必须完成以下动作：

1. 在对应产品需求文档中新增阶段需求编号。
2. 在 API 契约中补充接口。
3. 在数据模型文档中补充实体、字段或状态。
4. 在验收测试文档中补充测试矩阵。
5. 在风险合规文档中补充新风险和阻断项。
6. 明确是否影响 MVP 范围；默认不得影响已经锁定的 MVP 验收目标。
