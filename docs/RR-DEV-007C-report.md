# RR-DEV-007C 研究工作台项目抽屉修复报告

## 目标

修正 RR-DEV-007B 后仍不够准确的信息架构表达：既然项目列表是“研究工作台”的二级上下文，就不应单独放在左侧栏下方，也不应通过额外注释说明它属于工作台。

## 修改内容

- 将左侧“研究工作台”改为抽屉行：主体点击区域负责展开/收起，右侧 `+` 区域专门新增项目。
- 项目列表直接放在“研究工作台”下方，不再出现额外“项目”标题。
- `+` 不放在抽屉内容内部，避免和项目列表层级混在一起。
- 移除独立的 `project-section` 与重复的“研究工作台”说明。
- 保留左侧一级导航作为产品模块入口：研究工作台、雷达探索、论文追踪、知识库、消息中心。
- `雷达探索` 等未完成模块继续保持禁用和“待开放”状态。

## 信息架构约定

- 一级：左侧导航中的产品模块。
- 二级：当前一级模块内部的上下文或资源。
- Phase 1 中，项目列表属于“研究工作台”的二级资源，因此只在“研究工作台”抽屉内展示。

- URL：`http://127.0.0.1:3000`

- 再次点击 `研究工作台` 后项目列表展开，`aria-expanded=true`。
- 点击 `研究工作台` 右侧 `+` 后打开“新增研究项目”弹窗。

- 项目列表文本中不存在额外“项目”标题层。
- 页面中不存在旧的 `.project-section`。
- 页面中不存在旧的 `.section-parent`。
- 顶栏搜索仍为 0 个输入框。
- 无 Next error overlay。
