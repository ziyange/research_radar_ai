# 08 验收与测试

版本：v0.1  
日期：2026-06-14  
状态：MVP 基线

## 1. 验收原则

1. 所有 MVP 功能按需求编号验收。
2. 没有验收标准的功能不得进入开发。
3. 没有测试覆盖的功能不得标记完成。
4. AI 输出必须同时做结构校验和人工样本评测。
5. 推荐准确度是 MVP 第一验收目标。

## 3. 端到端验收场景

### E2E-001 冷启动到首日诊断

关联需求：`RR-MVP-002`、`RR-MVP-003`、`RR-MVP-006`、`RR-MVP-007`、`RR-MVP-008`。

步骤：

1. 用户注册并登录。
2. 创建研究项目。
3. 输入一句研究方向。
4. 系统生成画像草稿。
5. 用户修改关键词并确认画像。
6. 系统生成首日诊断。

通过标准：

- 画像结构完整。
- 首日诊断包含 3 篇高相关论文、2 篇方法迁移论文、1 个研究空白候选。
- 所有推荐都有解释。

### E2E-002 每日推荐与反馈纠偏

关联需求：`RR-MVP-009` 至 `RR-MVP-019`。

步骤：

1. 系统执行每日检索。
2. 多数据源返回候选论文。
3. 系统标准化、排重、推荐排序。
4. 用户标记若干无关论文和方法可借鉴论文。
5. 下一批推荐结果发生可解释变化。

通过标准：

- 排重后结果数可追踪。
- 用户反馈写入 UserFeedback。
- 新推荐绑定新的画像或偏好版本。

### E2E-003 AI 研读与知识库沉淀

关联需求：`RR-MVP-020` 至 `RR-MVP-025`、`RR-MVP-030`、`RR-MVP-031`。

步骤：

1. 用户对推荐论文触发快速分析。
2. 用户对其中一篇触发标准研读。
3. 系统记录成本。
4. 用户收藏论文，添加标签和备注。
5. 用户在知识库搜索该论文。

通过标准：

- AI 输出结构完整。
- 关键结论有事实分级。
- 成本记录归属正确。
- 知识库搜索可找到该条目。

### E2E-004 日报与周报

关联需求：`RR-MVP-026` 至 `RR-MVP-029`。

步骤：

1. 推荐任务完成后生成日报。
2. 日报进入站内消息。
3. 日报通过邮件发送。
4. 一周后生成周报。

通过标准：

- 报告结构完整。
- 邮件发送状态可追踪。
- 用户可在站内查看历史报告。

## 4. AI 评测标准

### 4.1 推荐评测集

每个种子研究方向需包含：

- 10 篇高相关论文。
- 10 篇方法可借鉴论文。
- 10 篇背景引用论文。
- 10 至 20 篇无关或低相关论文。

人工标注字段：

- 相关等级。
- 推荐原因。
- 是否应排除。
- 是否适合标准研读。

最低通过标准：

- Top 10 推荐中高相关或方法可借鉴论文占比达到内部验收阈值。
- 无关论文进入 Top 10 的比例低于内部验收阈值。
- 推荐解释人工通过率达到内部验收阈值。

具体数值阈值由种子数据完成标注后在本文件补充，补充前不得声称推荐系统完成最终验收。

### 4.2 AI 分析安全评测

每轮发布前必须检查：

- 虚构 DOI 数量为 0。
- 不存在把 AI 推测标记为原文事实的高风险样本。
- 所有关键结论有事实分级。
- 证据不足时标记不可追溯。
- 分析输出通过 JSON schema。

### 4.3 事实分级样本

至少包含：

- 摘要中明确说明的事实。
- AI 对摘要的归纳。
- 基于两篇以上论文的对比。
- 没有直接证据但合理的推测。
- 面向用户课题的研究启发。

## 5. 成本验收

关联需求：`RR-MVP-030`、`RR-MVP-031`、`RR-MVP-032`。

必须验证：

- 每次模型调用记录 token、模型、用户、项目、任务、功能。
- 失败的模型调用也记录成本或失败原因。
- 额度扣减幂等。
- 额度不足时任务不执行或进入等待状态。
- 管理员可按日期、用户、项目、模型、功能汇总成本。

MVP 默认成本策略：

- 自动日报只对 Top N 候选执行快速分析，N 由后台配置。
- 标准研读必须用户主动触发。
- 全文深读不作为 MVP 默认功能。

## 6. 合规验收

关联需求：`RR-MVP-010`、`RR-MVP-015`、`RR-MVP-033`、`RR-MVP-035`。

必须验证：

- 数据源适配器只调用开放 API 或公开数据。
- 不保存学校账号、图书馆密码或长期原始 Cookie。
- 开放全文链接记录来源和许可状态。
- 用户上传文件只用于用户授权项目。
- 日志不包含敏感凭证。

## 7. 发布前检查清单

发布前必须全部通过：

- 需求编号全部有测试项。
- API 契约与实现一致。
- 数据模型迁移与文档一致。
- 至少一个完整 E2E 冷启动流程通过。
- 至少一个每日推荐 E2E 流程通过。
- AI 输出 schema 校验通过。
- 成本记录可查询。
- 任务失败可重试。
- 合规检查无阻断项。
